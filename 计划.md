## 2. 后续 Agent 的使用规则

1. 开始任务前先读 [`AGENTS.md`](./AGENTS.md)，再查看本文件中的当前阶段和前置条件；需要理解产品取舍时读 [`HANDBOOK.md`](./HANDBOOK.md)。
2. 同一时刻只把一个里程碑标记为“进行中”。一个里程碑可以拆成多次改动，但不得跳过尚未满足的决策门。
3. 用户直接指定功能时，先将其映射到对应里程碑。如果前置条件未完成，只补齐所需的最小前置能力，不顺带重构无关模块。
4. 素材或渲染器未确定时，继续推进语义协议、调度、窗口能力、设置、测试和 Mock Agent，不继续扩大 SVG 专属动画代码。
5. 只有代码、测试和必要的真实 Tauri 验证均完成后，才能把阶段标为“已完成”。只有类型、空组件或占位命令不算完成。
6. 完成阶段后同步更新本文件的状态与实施备注、`HANDBOOK.md` 的真实产品状态；只有工程入口、规则或验证流程变化时才修改 `AGENTS.md`。
7. 如果实际实现推翻了这里的文件名或技术假设，优先保持职责与接口语义，并在同一次改动中修订本计划。

## 3. 总体架构目标

LLM、定时器、用户操作和未来的系统感知都不能直接操作 SVG 节点、Canvas、窗口坐标或 Tauri API。它们只能提交语义动作，由统一运行时校验和执行。

```text
用户 / Mock Agent / LLM / 番茄钟       系统观察 / 外部插件
                │                            │
                │                     ObservationEvent
                │                    schema / 来源 / 隐私 / 频率
                │                            │
                └──────── 核心策略映射 ────────┘
                              │
                              ▼
                       ActionRequest
                              │
                    权限、参数与安全策略校验
                              │
                              ▼
                      BehaviorScheduler
                 队列 / 优先级 / 中断 / 冷却 / 超时
                              │
                              ▼
                    CharacterController
                  ┌───────────┼───────────┐
                  ▼           ▼           ▼
 CharacterRenderer  WindowController  CapabilityController
 SVG/Rive/Spine     移动/显示/置顶    语音/计时/服装/感知/插件
```

长期边界：

- 模型只表达“招手”“看向用户”“移动到屏幕右侧”“换上某服装”，不生成 DOM selector、原始像素轨迹或 Tauri 命令。
- 渲染器只负责角色表现，不决定权限、冷却、窗口移动或产品策略。
- React 组件负责展示和交互编排，不承担长期调度器、业务时钟或原生状态的唯一事实来源。
- Rust 层负责必须依赖操作系统、可靠计时、持久化或原生权限的能力。
- 插件只产生版本化 `ObservationEvent` 或受限意图，不直接产生可信 `ActionRequest`；核心根据用户授权和本地规则决定是否转成动作。
- 不把 Codex、Claude Code、IDE 或特定媒体播放器写进核心业务。它们是可替换的事件来源适配器，卸载后基础桌宠仍完整可用。

## 4. 稳定领域契约

M1 至 M3 完成后，以下概念应成为后续功能共同依赖的最小核心。

### 4.1 动作请求与结果

动作使用 TypeScript 判别联合，并提供运行时校验。不能只依赖编译期类型，因为 LLM 和外部配置提供的是不可信 JSON。

```ts
type ActionSource = "user" | "agent" | "timer" | "system" | "dev";

interface ActionEnvelope<TType extends string, TPayload> {
  id: string;
  type: TType;
  payload: TPayload;
  source: ActionSource;
  requestedAt: number;
  timeoutMs?: number;
  correlationId?: string;
}

type ActionStatus =
  | "completed"
  | "interrupted"
  | "rejected"
  | "timed_out"
  | "failed";

interface ActionResult {
  actionId: string;
  status: ActionStatus;
  startedAt?: number;
  finishedAt: number;
  reason?: string;
  errorCode?: string;
}
```

第一版动作保持克制：

| 动作                             | 语义                | 执行者               |
| ------------------------------ | ----------------- | ----------------- |
| `motion.play`                  | 播放一次命名动作，如 `wave` | CharacterRenderer |
| `expression.set`               | 设置命名表情和可选持续时间     | CharacterRenderer |
| `look.set`                     | 看向归一化方向或语义目标      | CharacterRenderer |
| `window.move`                  | 移动到经过安全校验的语义目标    | WindowController  |
| `outfit.equip`                 | 切换指定服装包           | OutfitController  |
| `speech.say`                   | 说出给定文本，可被中断       | SpeechController  |
| `timer.start`                  | 启动可靠计时任务          | TimerController   |
| `timer.pause` / `timer.cancel` | 控制计时任务            | TimerController   |
| `wait`                         | 在动作序列中等待          | BehaviorScheduler |

首版不得加入“执行 JavaScript”“调用任意命令”“修改任意 CSS”一类通用逃生口。错误码应稳定，例如 `unsupported_action`、`invalid_payload`、`user_override`、`cooldown_active`、`permission_denied` 和 `renderer_unavailable`；调用方不得靠解析自然语言错误判断流程。

### 4.2 渲染器接口

接口表达能力，不暴露引擎对象：

```ts
interface CharacterRenderer {
  getCapabilities(): RendererCapabilities;
  playMotion(name: string, options?: MotionOptions): Promise<void>;
  setLookDirection(x: number, y: number): void;
  setExpression(name: string, options?: ExpressionOptions): Promise<void>;
  equipOutfit(outfitId: string): Promise<void>;
  reset(reason: ResetReason): void;
  dispose(): void;
}
```

- 视线使用 `[-1, 1]` 归一化坐标，不把 DPI 和素材像素泄漏给调用者。
- 每种渲染器返回能力清单；不支持的动作必须明确降级或拒绝。
- `dispose()` 清理事件、动画帧、计时器和运行中的动作。
- 点击命中也由渲染器提供。SVG 可使用 DOM 图形命中；Canvas 应使用多边形、命中遮罩或引擎命中测试，不能继续依赖 `elementsFromPoint()`。

### 4.3 调度通道与优先级

| 通道                | 示例         | 互斥规则               |
| ----------------- | ---------- | ------------------ |
| `locomotion`      | 自主移动、用户拖动  | 用户拖动永远打断自主移动       |
| `body-motion`     | 招手、睡觉、互动动作 | 同时只运行一个主体动作        |
| `gaze-expression` | 视线、眉毛、表情   | 可与主体动作并行，动作可临时接管   |
| `speech`          | TTS、说话状态   | 新语音按策略排队或打断旧语音     |
| `outfit`          | 换装、素材重载    | 与主体动作互斥，完成后恢复合法状态  |
| `timer`           | 番茄钟        | 后台独立运行，只在提醒时提交表现动作 |

默认优先级：安全停止 > 用户直接操作 > 设置/菜单命令 > 计时提醒 > Agent 主动行为 > idle。菜单打开、用户拖动和应用退出期间不得启动自主移动或换装。

状态：已具备，作为后续回归基线，不在此阶段继续扩展 SVG 动画。

必须保留：`400×500` 透明小窗口；呼吸、眨眼、视线、头发、耳朵和招手；非阻塞拖动；轮廓外穿透；原生右键菜单、键盘菜单和可靠退出。

目标：建立不依赖 React、Tauri 和具体渲染器的动作类型与运行时校验。

前置条件：M0 可构建、可测试。

交付物：

- [x] 能力不足返回稳定结果，不把适配器异常直接泄漏给调用方。

## 6. 跨阶段质量轨道

### 测试

- 纯数学、协议、状态机和策略优先做无 UI 单元测试。
- 适配器使用 fake renderer、fake clock、fake provider 和 fake window 做集成测试。
- 插件协议使用 fake plugin、畸形事件、版本不兼容和事件洪泛测试；CI 不依赖真实 Codex、Claude Code 或媒体播放器。
- 菜单、窗口、DPI、穿透、多屏和全局钩子必须真实 Tauri 验证。
- LLM 测试默认用确定性 Fake Model，不让 CI 调用付费 API。

### 可观测性与性能

- 动作、Agent turn、计时和原生命令使用可关联 ID。
- 日志记录状态与错误码，不记录 key、完整聊天或敏感感知数据。
- 更换渲染器或增加常驻能力前后记录空闲 CPU、内存和启动时间。
- 不在鼠标高频事件中触发全树重渲染、同步磁盘写入或无界 IPC。
- 所有动画帧、监听、计时器和后台任务都有确定 cleanup。

### 安全、隐私与打扰控制

- capability 最小授权；Agent 工具白名单化，不设计任意执行后门。
- 第三方插件默认不可信，使用声明式 manifest、进程外隔离、事件 schema、权限开关与频率预算；插件不可继承主程序 API key 或原生 capability。
- 感知和外发默认关闭或最小化，并有用户可见总开关。
- 秘密与仓库、普通日志和普通设置隔离。
- 保留键盘菜单、退出和设置路径；动画尊重 reduced motion。
- 自主移动、语音、通知和主动对话有频率限制、暂停和即时中断。
