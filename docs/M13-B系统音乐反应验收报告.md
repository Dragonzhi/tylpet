# M13-B 系统音乐反应验收报告

> 状态：代码、自动化检查和用户真实 Windows 播放器验收均已完成（2026-07-23）。

> 范围：系统媒体播放状态、观察事件接入、持续视觉反应、设置、安全停止和隐私边界。不包含媒体控制、标题/歌手/歌词、音频采样、节拍检测或第三方插件。

## 1. 数据边界

原生观察线程只执行以下读取链路：

1. `GlobalSystemMediaTransportControlsSessionManager::GetCurrentSession()`。
2. `session.GetPlaybackInfo()`。
3. `PlaybackInfo.PlaybackStatus()`。

最终只向主窗口发送 `playing | paused | stopped`。实现没有调用 `TryGetMediaPropertiesAsync()`、`SourceAppUserModelId()`、时间线 API、音频捕获或播放器控制 API。

## 2. 行为链路

```text
Windows 当前媒体会话
  → system-media-playback
  → ObservationEvent(media.playback)
  → 来源/时效/去重/频率/安静时段校验
  → media.react
  → BehaviorScheduler(ambient, agent priority)
  → SvgCharacterRenderer
  → data-media-playback + CSS 渐变律动
```

- `playing`：轻微整体起伏、左右马尾和耳朵错相摆动，并显示克制的小粒子。
- `paused`：目标幅度在约 `650ms` 内收束到 0。
- `stopped` 或无可用媒体会话：恢复原待机动画。
- 媒体变化不触发招手、鞠躬等一次性动作，也不会抢占身体动作通道。

## 4. 真实窗口验收

1. 运行 `npm run tauri dev`，打开设置。
2. 开启“允许外部状态触发角色反应”和“对系统音乐作出反应”。
3. 使用能出现在 Windows 媒体面板中的播放器播放音乐，例如系统媒体会话兼容的桌面播放器或浏览器媒体页面。
4. 开始播放后约一秒内，小洛宝应出现轻微整体、马尾、耳朵和粒子律动；普通呼吸、鼠标跟随和单击招手仍可用。
5. 暂停音乐，律动应平滑收束而不是突然跳回；继续播放后应渐入恢复。
6. 停止播放或关闭当前媒体会话，角色应恢复原待机，不残留粒子或持续偏转。
7. 把“音乐反应强度”依次调到 `0%`、`50%`、`100%`，确认 0% 无反应，其余幅度逐级变化且无需重启。
8. 将安静时段设置为覆盖当前时间，确认正在播放的律动会停止；移出当前时间范围后重新检测当前播放状态。
9. 点击“立即停止所有自主行为”，确认律动停止；关闭再开启外部反馈总开关后才能恢复。
10. 在 Windows“设置 → 辅助功能 → 视觉效果”关闭动画效果，确认媒体观察与律动关闭；恢复系统动画后可以重新响应。
11. 开发控制台的“模拟媒体播放/暂停/停止”可独立验证前端链路，但不能替代真实 Windows 媒体会话验收。

## 5. 已知边界

- 首版跟随 Windows 选择的“当前媒体会话”。多个播放器同时存在时，不自行读取应用名来做选择。
- 未接入 System Media Transport Controls 的播放器不会被识别，这是预期降级，不会改用录音或音频波形捕获绕过。
- 当前是固定节奏的视觉反应，不分析真实 BPM。真实节拍分析若未来加入，必须另立音频权限和隐私评审。
