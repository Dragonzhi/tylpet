## 1. P0 要回答的问题

P0 不是开始制作完整动画编辑器，而是用最小代码回答一个会决定整个 M8 方向的问题：

> `@svgedit/svgcanvas` 能否稳定承载 `小洛宝.glax.svg`，让我们选中语义部件、围绕已有 pivot 预览两关键帧动作，并把动作数据保存后重新打开？

只有以下结果全部成立，才进入 P1：

1. 导入后视觉、ID、分组和 pivot 没有不可接受的漂移。
2. 能以 `arm_right` 为一个整体选择，而不是误拆内部路径。
3. 能围绕素材中的右手 pivot 预览两关键帧旋转补间。
4. 导出的实验动作 JSON 可以重新导入并得到相同结果。
5. `src/assets/小洛宝.glax.svg` 和原始 `小洛宝.svg` 都不被编辑器重写。
6. 长期代码不调用 `getPrivateMethods()` 或依赖 SVG-edit 完整编辑器 UI。

P0 失败不是项目失败。失败时先记录阻塞点，再决定替换画布内核、只借用更少的 svgcanvas 能力，或完全使用浏览器原生 SVG DOM；不能在失败基础上继续堆时间轴 UI。

### 1.1 P0 的双样例策略

P0 默认加载 `src/assets/小洛宝.glax.svg`。它仍是普通 SVG，只是历史上为
Glaxnimate 兼容而生成：153 个带复杂 transform 的矩形已转换为路径并把变换烘焙
进坐标，语义分组、50 个唯一 `inkscape:label`、207 个唯一 ID、pivot 和 viewBox
均得到保留。

`src/assets/小洛宝.svg` 继续作为 Inkscape 美术源和对照样例。P0 不用 `.glax.svg`
覆盖或重命名原文件，而是执行：

1. `.glax.svg`：主要编辑器输入和动作预览样例。
2. 原始 `.svg`：导入诊断、结构差异和视觉回归对照。

现有 676×1068 Inkscape 渲染对比中，两者有约 `0.214%` 像素不同，平均通道绝对
误差约 `0.0033`，肉眼一致；差异主要位于路径化后的抗锯齿边缘。P0 报告仍需
重新记录测试环境和截图，不能仅引用本次预检查。

```text
  vite 7.3.6
  vitest 3.2.7
```

这些依赖安装在 `tools/motion-editor/`，不能写入桌宠根 `package.json`。

### 3.2 本机已有即可

- Git。
- Node 22 与 npm。
- Edge 或 Chrome，用于真实编辑器验证。
- Inkscape，仅在需要对照素材或修改 SVG 分组时使用；它不是编辑器运行依赖。

### 3.3 P0 不安装

- 完整 `svgedit` npm 包。
- SVG-edit 仓库源码或完整编辑器 UI。
- Electron、第二套 Tauri 壳或 Rust 插件。
- Rive、Spine、Live2D SDK。
- 数据库、云服务、Docker、Python 图形库。
- Playwright、状态管理库、拖拽库和 UI 组件库。
- SVGO 自动优化流程；它可能改写 ID、矩阵或结构，P0 禁止自动处理生产 SVG。

Playwright 只在 P2/P4 出现稳定 UI 后再评估；P0 用 Vitest 覆盖纯逻辑，并进行一次真实浏览器验收。

## 6. P0 代码边界

### 6.1 `SvgCanvasAdapter`

所有上游调用集中在一个适配器中，React 组件不能散落调用 svgcanvas：

```ts
interface StageAdapter {
  mount(container: HTMLElement): void;
  loadSvg(source: string): ImportResult;
  selectPart(partId: string): boolean;
  applyPreviewTransform(
    partId: string,
    transform: PreviewTransform,
  ): void;
  restoreBindPose(partId: string): void;
  dispose(): void;
}
```

这里的公开调用键必须是语义 `partId`，例如 `arm_right`，不能是
`layer21`、`rect3-...` 之类的素材 DOM ID。适配器在导入阶段建立内部索引：

```ts
interface ImportedPartRef {
  partId: string;
  inkscapeLabel: string;
  sourceElementId: string;
  element: SVGElement;
}

type PartIndex = Map<string, ImportedPartRef>;
```

当前素材的推荐解析流程：

1. 从原始 SVG 读取 `inkscape:label`，将唯一且符合命名规则的 label
   归一化为 `partId`。
2. 同时记录该节点当前的 DOM ID，例如 `arm_right -> layer21`。
3. SVG 载入 svgcanvas 后，用当前 DOM ID 找到工作副本中的元素，并把元素引用
   缓存在 `PartIndex`。
4. React、时间轴和动作 JSON 始终只传 `partId`；只有 adapter 知道 DOM ID。

svgcanvas 的 `selectOnly()` 接收的是 `Element[]`，因此上层不需要按 DOM ID
调用选择 API。内部 DOM ID 仍用于在导入前后的两棵 DOM 之间重新定位元素，必须
诊断缺失、重复或载入后被改写的 ID。

P0 不要求用户修改生产 SVG，也不把 `data-part` 作为前置条件。若 P0 证明
`inkscape:label -> DOM ID -> 工作副本 Element` 的映射稳定，P1 再冻结绑定格式。
长期建议让 sidecar rig 持有语义 `partId` 与素材绑定；如确实需要 `data-part`，
由导出器写入派生的运行时 artwork，而不是静默回写 Inkscape 源文件。

P0 允许使用并记录的 `@svgedit/svgcanvas` 公开入口：

- `new SvgCanvas(container, config)`
- `setSvgString()`
- `getSvgContent()` / `getSvgRoot()` / `getSvgString()`
- `selectOnly()` / `clearSelection()`
- `changeSelectedAttributeNoUndo()`，仅用于临时预览且必须恢复
- `bind()` / `unbind()`
- `setZoom()` / `getZoom()`

禁止：

- `getPrivateMethods()`。
- 复制 SVG-edit editor UI 或依赖 extension 私有对象。
- 把 svgcanvas DOM、selection 对象或 undo command 写进项目文件。
- 让 P0 实验 JSON 成为正式运行时 schema。

### 6.2 P0 的只读与可编辑边界

P0 是“源素材只读、动作数据可编辑”，不是整个页面完全不可编辑：

| 对象 | P0 是否允许修改 | 是否持久化 |
|---|---|---|
| `src/assets/小洛宝.svg` | 否 | Inkscape 美术源，永不由 P0 写回 |
| `src/assets/小洛宝.glax.svg` | 否 | P0 默认输入，永不由 P0 写回 |
| svgcanvas 中的 SVG 工作副本 | 仅允许预览期临时 transform | 否，停止或卸载时恢复 |
| Part 的路径、矩形、颜色和分组 | 否 | 否 |
| pivot / bind pose | P0 只读取和显示 | 写入实验 JSON 快照，不回写 SVG |
| rotation 关键帧、帧号、循环和缓动 | 是 | 写入实验 motion JSON |
| 通用时间轴和 SVG 几何编辑 | 否 | 留到后续阶段重新评估 |

`changeSelectedAttributeNoUndo()` 只作用于 svgcanvas 创建的内存工作副本。载入时
必须保存目标元素原始 `transform` 属性的“存在/不存在状态和值”，每一帧都从 bind
transform 重新计算预览结果，不能在上一帧矩阵上累乘。暂停、停止、切换 Part、
重新导入、HMR 和 `dispose()` 都必须精确恢复原值。

P0 导出的只有诊断和实验 motion JSON，不导出修改后的 artwork。P2 所说的舞台
拖动/旋转/缩放，默认也是编辑关键帧增量，而不是修改路径节点或重写源 SVG；若未来
要增加几何编辑模式，必须作为单独功能重新设计保存和撤销边界。

### 6.3 React 生命周期

- svgcanvas 实例只在 adapter 的 `mount()` 创建一次。
- `dispose()` 必须解绑事件、恢复预览变换并清空容器。
- 编辑器入口暂不启用 React StrictMode；先证明上游实例可可靠销毁，再决定是否开启。
- HMR 期间不得产生第二个 canvas、重复快捷键或遗留选框。

### 6.4 素材读取

- 默认样例读取 `src/assets/小洛宝.glax.svg` 的文本，不复制或覆盖该文件。
- 提供“载入原始对照”入口读取 `src/assets/小洛宝.svg`，输出同一套诊断和视觉对比。
- 同时提供 `<input type="file" accept="image/svg+xml">`，验证普通用户导入路径。
- Vite 仅为默认样例允许读取仓库素材目录；不能开放任意本机目录 HTTP 访问。
- 原始 SVG 文本和 SHA-256 指纹在载入时保存，P0 结束时再次比对。

## 7. P0 分步实施

### P0-A：脚手架和上游探针

交付：

- 隔离包、lockfile、固定端口和基本页面。
- `SvgCanvasAdapter` 能创建与销毁 canvas。
- 页面显示 svgcanvas 版本、源素材指纹和错误日志。

验收：连续 HMR 三次只有一个舞台实例；关闭页面后无未处理异常。

### P0-B：安全导入与诊断

导入前使用浏览器 `DOMParser` 检查：

- XML/SVG 解析错误。
- `script`、`foreignObject`、`on*` 事件属性。
- 外部 `href`、`xlink:href`、CSS `url(http...)`。
- 重复 ID、无 ID 的候选语义组。
- `pivot_*` 标记及其对应部件。
- 无法求逆的 matrix。

P0 只诊断并拒绝危险输入，不自动“修好”生产 SVG，也不做全文件格式化。

验收：页面给出可复制的结构化报告；现有素材没有被静默删节点或改 ID。

### P0-C：语义选择和 pivot

- Part 列表按稳定 ID 展示，pivot 默认隐藏但可在调试开关下显示。
- 点击 `arm_right` 时选中整个语义组。
- 将 `pivot_arm_right` 换算到 `arm_right` 预览使用的坐标系。
- 显示当前 Part、pivot 数值、bind transform 和世界矩阵。

验收：在 100% 与 150% 缩放下，点击、选择框和旋转中心一致。

### P0-D：两关键帧动作

固定一个实验 clip：

```text
id: p0-wave
part: arm_right
fps: 24
frame 0:  rotation 0deg
frame 12: rotation -55deg
frame 24: rotation 0deg
easing: easeInOut
```

UI 只需要：播放/暂停、帧滑块、三个关键帧数值输入和循环开关。此阶段不做通用时间轴。

每帧流程：

1. 根据播放时间计算 frame。
2. 纯函数插值得到 rotation。
3. 以 pivot 组合预览矩阵。
4. 临时应用到舞台元素。
5. 停止、切换素材和卸载时恢复 bind pose。

验收：动作至少循环十次无累计漂移；暂停在任意帧后重播结果一致。

### P0-E：导出、重开与往返

实验 JSON 必须显式标记不可用于生产：

```json
{
  "experimentalSchema": "m8-p0@1",
  "productionReady": false,
  "sourceFingerprint": "sha256:...",
  "clip": {
    "id": "p0-wave",
    "partId": "arm_right",
    "fps": 24,
    "durationFrames": 24,
    "pivot": { "x": 0, "y": 0 },
    "keyframes": []
  }
}
```

- 导出使用浏览器下载。
- 重开使用文件输入，不引入本地原生文件插件。
- 序列化字段顺序稳定。
- 导出前后不得调用 `getSvgString()` 覆盖生产 SVG。

验收：导出、刷新页面、重新导入后关键帧、pivot、插值结果和源指纹一致。

### P0-F：结论报告

创建 `docs/M8-P0验收报告.md`，必须记录：

- 实际安装版本与 lockfile 状态。
- SVG 导入诊断摘要。
- `.glax.svg` 与原始 SVG 的结构、视觉差异，以及派生文件的可复现生成方式；若暂时无法复现，明确记录为进入生产前的阻塞项。
- 视觉、选择、pivot、补间和重开结果。
- 使用的 svgcanvas 公开 API。
- 是否存在必要 patch；若有，列出文件、原因和替代方案。
- 已知差异截图或复现步骤。
- 明确结论：`通过`、`有条件通过` 或 `失败`。

只有报告为“通过”或阻塞项有明确小范围解决方案的“有条件通过”，才把 `计划.md` 的 P0 标为完成并进入 P1。

## 8. 自动化测试清单

P0 至少包含：

1. 危险 SVG 节点和外部引用被拒绝。
2. 重复 ID、缺失 pivot、孤立 pivot 能被诊断。
3. 关键帧首帧、末帧、中间帧和循环边界正确。
4. `easeInOut` 输出有限且不越过端点。
5. pivot 旋转矩阵在 0 度时为单位增量。
6. 实验项目 JSON 导出再导入深度等价。
7. 未知版本、未知 part、非有限数值和越界帧被拒绝。

测试不能依赖真实时间；播放时钟与 `requestAnimationFrame` 分开，插值使用显式 frame 输入。

## 10. 停止条件

遇到下列任一项，停止扩展 UI并先写报告：

- svgcanvas 导入后不可预测地拆组、改 ID 或丢失 pivot。
- 必须广泛调用 `getPrivateMethods()` 才能完成选择或预览。
- 每次 `getSvgString()` 都产生大范围、不可解释的结构重写。
- 带 matrix、镜像或斜切的部件无法稳定围绕 pivot 预览。
- HMR 或重新载入会留下重复舞台和事件监听。
- 为完成右手两关键帧就需要实现通用时间轴、完整撤销栈或修改生产渲染器。

这些条件说明 P0 已获得有价值的否定结论，应回到画布内核决策，而不是继续投入沉没成本。

## 11. P0 之后的交接

P0 通过后才开始 P1，并做以下变化：

- 删除 `experimentalSchema`，冻结正式 `rig.json` / `motions.json` schemaVersion 1。
- 把插值、矩阵、校验和稳定序列化迁入独立的纯 TypeScript 共享核心。
- 编辑器与 `SvgCharacterRenderer` 使用同一核心和测试向量。
- P2 再实现通用时间轴、F6、复制粘贴、缓动 UI 和统一撤销。
- P3 再修改桌宠生产运行时；P0/P1 不替换现有 CSS 招手动画。

这样即使 P0 的画布技术需要更换，动作格式、纯数学和桌宠行为架构也不会被一起推倒。
