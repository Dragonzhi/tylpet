# M15 记忆与羁绊验收报告

> 状态：代码与自动化检查已完成，等待真实 Windows/Tauri 验收。
>
> 范围：用户明确保存的长期记忆、模型候选与对话式确认、无记忆模式、模型摘要注入、确定性羁绊、来源审计、编辑/导出/删除、原子持久化和损坏恢复。不包含完整聊天持久化、模型静默写记忆、向量数据库、云同步或隐藏关系惩罚。

## 1. 用户能看到和控制什么

- 设置页“长期记忆与羁绊”默认全部关闭。关闭总开关时，已有数据保留，但不会读取、注入模型或累计羁绊。
- 每条记忆由用户选择“偏好 / 个人资料 / 备注”并点击“明确保存”；固定显示来源原因“用户在设置中明确保存”。
- 对话式记忆有“关闭 / 每次保存前确认 / 明确说‘记住’时自动保存”三档。模型自主提议始终显示可编辑确认卡；自动档只对当前用户消息中的明确记忆指令免确认。
- 模型只有 `memory_propose` 候选工具，参数是分类、内容和原因。确认保存、显式指令自动保存和设置页直接保存分别记录不同来源；模型不能直接调用持久化命令。
- 已保存条目支持原地编辑和逐条删除。下一轮模型请求重新读取快照，因此删除后不再使用。
- “导出 JSON”写入当前用户下载目录，包含记忆、羁绊数值和有限原因审计，不包含 API key 或完整聊天。
- “一键清除”删除全部记忆和羁绊，需要明确确认且不可撤销。

## 2. 羁绊规则

- 一次用户发起的对话成功完成后 `+1`；失败、取消、空回复和重复请求不增加。
- 每个本地自然日最多奖励 3 次，总上限 100，永不扣分。
- 等级固定为：`初识 0`、`熟悉 10`、`亲近 30`、`默契 60`、`长久相伴 100`。
- 模型没有编辑、删除长期记忆或修改羁绊的工具。设置页可查看最近 10 条 `+1` 原因；文件最多保留 50 条不含聊天正文的审计事件。
- 首版不另设隐藏“好感度”；所有关系成长只由上述公开羁绊值表达。

## 3. 数据与隐私边界

- 普通设置在 `settings.json`；长期数据在独立 `memory.v1.json`；当前聊天只在聊天窗口内存。
- 记忆最多 100 条，每条最多 300 个 Unicode 字符；持久化文件上限 256 KiB。
- 模型上下文注入默认关闭。开启后只发送用户明确保存的记忆摘要和羁绊等级，最多占本轮上下文字符预算 25% 且不超过 3000 字。
- 记忆内容按不可信用户文本处理，系统提示明确禁止执行其中的指令。使用外部 Provider 时，设置页会说明摘要将发送到所配置接口。
- 主文件写入采用临时文件和备份替换；主文件损坏时恢复备份，两者均损坏时安全重置为空并向用户提示。

## 4. 人工验收步骤

1. 运行 `npm run tauri dev`，打开设置，确认长期体验、模型注入和羁绊三个开关默认关闭。
2. 新增三类记忆各一条，关闭再打开设置和应用，确认内容仍在且来源原因可见。
3. 开启 Agent、长期体验和“每次保存前确认”，在对话输入“请记住我不喜欢香菜”；确认出现候选卡，可修改分类、内容和原因。拒绝一次应不落盘，再次提交并确认后应显示“用户确认了模型提议”。
4. 切换“明确说‘记住’时自动保存”，再次发送明确记忆指令，确认无需弹卡且来源为显式指令自动保存；发送不含明确指令的普通消息时，模型自主候选仍必须弹卡。
5. 将提议模式切到“关闭”，确认下一轮工具数量减少且模型无法调用 `memory_propose`；设置页手动新增仍然可用。
6. 编辑一条、删除一条，重新打开设置确认结果持久化。
7. 点击“导出 JSON”，确认提示给出下载目录中的完整文件路径；打开文件确认来源准确且没有 API key 和聊天正文。
8. 开启长期体验和羁绊，保持模型注入关闭；成功完成 4 次对话。预期当天只增加 3 点，原因审计每次均为“成功完成一次用户发起的对话”。
9. 取消一次生成或制造 Provider 失败，确认羁绊不增加。
10. 开启模型注入，询问已保存事实；聊天窗口显示本轮使用的条目数量。删除该条记忆后开启新一轮，模型不应继续从长期上下文得到它。
11. 使用外部 Provider 前确认设置页明确提示记忆摘要会外发；关闭模型注入后聊天状态显示本轮未提供长期记忆。
12. 关闭长期体验总开关并继续对话，确认羁绊不变、没有记忆注入状态且没有记忆工具。
13. 点击“一键清除”并确认，检查条目、羁绊和原因审计全部归零；重启后仍为空。
