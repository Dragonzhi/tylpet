# M13-C 创作者插件与开发 Agent 桥接验收报告

> 状态：代码与自动化检查完成后，等待真实 Tauri 侧载闭环验收。

> 范围：声明式 manifest、本地安装确认、启用/禁用/卸载、随机凭据、回环桥接、动态 grant、`ltypet emit`、Mock Host 和 `dev-agent-hooks` 示例。不包含在线市场、自动安装脚本或进程内第三方代码。

## 1. 安全边界

- 桥接只绑定 `127.0.0.1:0`，由系统分配随机端口，不监听局域网地址。
- 每个插件获得独立 256-bit 随机 token；主程序重启时凭据文件会以新回环地址重写。
- manifest 严格校验插件 ID、`x.y.z` 版本、宿主范围、声明事件和最高敏感级别；未知字段、入口和事件会拒绝。
- v1 开发 Agent payload 只能包含一个白名单 `state`，请求上限 8 KiB，每插件每分钟最多 20 个事件。
- 禁用会让原生身份立即拒绝请求并撤销前端 grant；卸载还会删除凭据目录。
- 凭据位于当前用户应用数据目录。同一 Windows 用户下运行的恶意程序仍处于同一信任边界，因此只侧载可信 hook。

## 3. 真实 Tauri 验收

1. 运行 `npm run tauri dev`，确认小洛宝、托盘和退出路径正常。
2. 在设置中开启“允许外部状态触发角色反应”，进入“创作者插件”。
3. 输入 `examples\plugins\dev-agent-hooks\ltypet.plugin.json` 的绝对路径并点击“检查 manifest”。
4. 确认界面显示插件 ID `dev-agent-hooks`、事件权限仅 `dev-agent.status`、敏感级别 `status`；此时插件列表尚未出现，证明检查不等于安装。
5. 点击“确认安装”，复制插件卡片显示的 `credential.v1.json` 完整路径。
6. 在另一 PowerShell 中调用：

```powershell
examples\plugins\dev-agent-hooks\emit.ps1 -State completed -Credential "<上一步的凭据路径>"
```

7. CLI 应返回“事件已提交”，小洛宝应通过现有语义映射招手；依次测试 `waiting_for_user`、`failed`，确认分别走既有眨眼/鞠躬映射，且 hook 未传 motion ID。
8. 关闭插件“启用”开关，再执行相同命令；CLI 应明确返回“插件已禁用”，角色不动作。重新启用后恢复。
9. 关闭外部状态反馈总开关；CLI 可以提交到宿主，但 ObservationHost 不调度动作。重新开启后恢复。
10. 连续快速提交 21 次，确认达到每分钟预算后请求被拒绝，桌宠本身仍可交互和退出。
11. 卸载插件后再次调用旧凭据路径，命令应因凭据文件已删除而失败；设置页不再显示插件。
12. 修改示例 manifest：把入口改为 `javascript`、事件改为 `shell.output` 或宿主范围改为 `>=1.0.0 <2.0.0`，确认“检查 manifest”阶段即拒绝。
13. 重装插件后重启桌宠，不应丢失安装记录；凭据路径不变，hook 重新读取文件后仍可提交。

## 4. 通过门槛

- 安装前能看见权限差异，检查和确认是两个独立动作。
- 启用、禁用、重启、卸载与旧凭据失效路径符合预期。
- `dev-agent-hooks` 无需修改核心仓库即可提交生命周期状态并触发既有低优先级反应。
- 透明窗口、点击穿透、拖动、媒体反应、单击招手、对话、托盘和退出路径没有回归。
