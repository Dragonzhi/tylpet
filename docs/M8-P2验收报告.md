## 1. 结论

M8 P2 已完成，可以进入 P3「生产运行时接入」。独立编辑器已经能够在不修改源码和 JSON 的情况下加载小洛宝、管理 Clip、制作和编辑关键帧、预览动作、统一撤销，并按 v1 schema 导入及 canonical 导出 rig/motions。

本阶段没有修改桌宠的 Tauri 窗口、点击穿透、拖动、托盘或生产动作逻辑，也没有把 svgcanvas 引入生产运行时。

## 2. 交付内容

### 2.1 编辑器生产状态与历史

- `EditorDocument` 中的 `CharacterRigV1 + MotionLibraryV1` 是唯一生产事实；舞台 DOM 只做投影。
- 所有 Clip、关键帧、pivot 和 rig 修改经纯命令入口校验后提交。
- 统一历史支持最多 200 步撤销/重做、保存签名、脏状态和离开页面提醒。
- 连续 Gizmo 拖动只在 `pointerup` 生成一次历史记录；取消手势不污染文档。
- 关键帧批量移动和粘贴在目标冲突时整体拒绝，不产生半提交。

主要实现：

- `tools/motion-editor/src/editor/model/`
- `tools/motion-editor/src/editor/history/EditorHistory.ts`
- `tools/motion-editor/src/timeline/geometry.ts`

### 2.2 舞台、Part 与 Transform Gizmo

- Part 树按 rig 的逻辑父级和稳定绘制顺序展示，支持折叠、选择、临时隐藏和锁定。
- 舞台点击与 Part 树双向同步；锁定 Part 不再响应舞台命中。
- 项目自有 Gizmo 围绕 rig 中的真实 Part-local pivot 工作，支持移动、旋转、等比缩放、`scaleX`、`scaleY` 和 pivot 拖动。
- 支持 `Space + 拖动` 平移舞台；素材按视口适配时不改写源 Part transform 或导出 SVG。
- `renderSlot` 只重排语义 Part 的预览节点，清除关键值或撤销后恢复源绘制顺序。
- 适配层只使用 svgcanvas 的公开加载/选择能力；没有把私有 selector 或 svgcanvas history 作为生产数据来源。

主要实现：

- `tools/motion-editor/src/svgcanvas/SvgCanvasAdapter.ts`
- `tools/motion-editor/src/components/parts/PartTree.tsx`
- `tools/motion-editor/src/components/stage/TransformGizmo.tsx`
- `tools/motion-editor/src/components/inspector/TransformInspector.tsx`

### 2.3 Clip、时间轴与关键帧

- Clip 支持新建、复制、改名、删除，并可编辑 fps、持续帧数、循环和程序动画抑制。
- 时间轴提供帧标尺、播放头、Part/属性轨道、离散层级标记、水平缩放和标签/轨道纵向同步。
- 支持单选、多选、范围选择、拖动和键盘批量平移关键帧。
- `F6` 在当前帧插入/更新完整数值姿态；`Shift+F6` 删除当前关键帧。
- 支持复制、粘贴、删除以及 `Ctrl+Z`、`Ctrl+Shift+Z`/`Ctrl+Y`。
- Inspector 支持数值 transform/opacity、预设 easing、四参数 cubic-bezier 和 `renderSlot`。
- 播放使用时间戳推进，支持暂停、首尾帧、逐帧、一次播放和 repeat 循环。

主要实现：

- `tools/motion-editor/src/App.tsx`
- `tools/motion-editor/src/components/timeline/Timeline.tsx`
- `tools/motion-editor/src/styles.css`

### 2.4 共享数学与数据闭环

- 共享核心新增绕非零 pivot 的 authored transform 分解，可从 `bindMatrix⁻¹ × editedLocalMatrix` 恢复无 skew 的位移、旋转和缩放。
- 对奇异矩阵、非有限值和不受支持的 skew 明确拒绝。
- Rig 导入严格核对 schema、素材文件名、指纹、viewBox 和 source binding。
- Motions 导入继续按当前 rig 严格校验；导出使用共享包 canonical serializer。
- UI session、选择、隐藏/锁定、缩放、播放头、草稿和撤销栈均不进入 v1 JSON。

主要实现：

- `packages/character-motion/src/math/affine2d.ts`
- `tools/motion-editor/src/project/v1Project.ts`

## 4. 真实浏览器验收

通过 Chromium DevTools Protocol 在真实浏览器引擎中执行了以下操作，未观察到控制台错误：

- 加载 `小洛宝.glax.svg`：识别 33 Part、15 pivot，舞台自适应视口。
- 从 UI 创建 `idle`、`wave`、`bow`，每个 Clip 均可选择 Part 并插入关键帧。
- 旋转 Gizmo 后一次撤销恢复 identity；pivot 拖动、`scaleX` 和 `renderSlot` 同样一次撤销恢复。
- 在 120 帧密集时间轴中插入 100 个 marker，拖动、冲突拒绝、复制到末帧和撤销正常。
- `p0-wave` repeat 连续播放超过 10 秒，暂停后播放头保持稳定。
- 隐藏/锁定只影响编辑会话，不产生脏状态；锁定后舞台点击不再选中该 Part。
- `Space + 拖动` 可平移舞台，素材的生产 transform 不变。
- 真实 `xiaoluobao.rig.v1.json` 与 `src/assets/p0-wave.motions.v1.json` 均可从文件选择器导入。
- DPR 1、1.5、2 下围绕真实 pivot 旋转，最大测得误差小于 `0.011 CSS px`。

无头浏览器能够观察到 rig/motions 两个下载请求以及与 canonical 文本一致的字节数，但 Edge/Chrome 的无头下载环境会随后报告 `canceled`，未能把文件持久化到测试目录。因此操作系统文件落盘仍列为一次 GUI 人工复核项；序列化、严格解析和字节往返已由自动化测试覆盖。

## 5. 人工复核步骤

在仓库根目录运行：

```powershell
npm --prefix tools/motion-editor run dev
```

然后在 `http://127.0.0.1:1430` 完成：

1. 点击“初始化画布”与“载入角色”。
2. 新建 `idle`、`wave` 和一个自选动作，在不同帧选择部件并按 `F6`。
3. 使用舞台手柄和 Inspector 修改旋转、缩放、pivot、缓动与层级，确认一次撤销恢复一次手势。
4. 播放并逐帧检查三个动作。
5. 点击“导出项目”，确认得到一份 `.rig.v1.json` 和一份 `.motions.v1.json`。
6. 刷新编辑器，重新导入这两份文件，确认 Clip、关键帧、pivot、缓动和层级保持一致。

## 6. P3 交接边界

P3 只消费 v1 rig/motions：

- 通过现有 `SvgCharacterRenderer`/`CharacterRenderer` 契约播放命名动作。
- 处理编辑动作与视线、眨眼、呼吸、头发物理的通道叠加和抑制。
- 不把 svgcanvas、编辑器组件、DOM 引用、session、draft 或 history 带入桌宠生产运行时。
- 不在 P3 偷改 v1 schema；若发现格式不足，先形成独立 schema 变更决策。
