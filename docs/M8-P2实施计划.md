## 1. 结论与实施门

M8 P1 已通过，可以进入 P2。P2 的目标不是继续增加“预览按钮”，而是把当前只能载入、
播放和导出 JSON 的验证页面，变成一套可以实际制作动作的 Flash 风格 MVP。

P2 完成后，用户必须能在不修改源码和 JSON 的情况下：

1. 载入小洛宝素材和 rig。
2. 新建、复制、重命名和删除 Motion Clip。
3. 从 Part 树或舞台选择部件。
4. 在时间轴插入、修改、复制、粘贴、删除和拖动关键帧。
5. 通过属性面板和舞台手柄编辑位移、旋转、缩放、透明度、pivot、缓动和绘制槽位。
6. 播放、暂停、循环、逐帧检查动作。
7. 撤销和重做所有生产数据修改。
8. 导出通过 v1 schema 校验、重载后表现一致的 `rig.json` 和 `motions.json`。
9. 独立制作并保留 `idle`、`wave` 和一个自选动作。

只有以上完整闭环通过，才能把 P2 标为完成。做出时间轴外观、只有内存状态或只支持
既有 `p0-wave` 数据，不算完成。

## 2. 当前基线

P2 直接建立在以下已验证事实之上：

- 编辑器位于 `tools/motion-editor/`，与桌宠生产依赖和窗口隔离。
- `@svgedit/svgcanvas` 固定为 `7.4.2`，只通过适配层使用。
- `@ltypet/character-motion` 已冻结 rig/motions v1 schema，并提供校验、采样、矩阵、
  缓动和稳定序列化。
- `小洛宝.glax.svg` 当前可识别 33 个 Part、15 个 pivot，导入无错误。
- 当前 rig 使用真实素材指纹和真实局部 pivot；素材已经扁平化，因此现有
  `logicalParentId` 均为 `null`、`bindMatrix` 为 identity 是真实状态，不是占位数据。
- `src/assets/p0-wave.motions.v1.json` 可以完成导入、预览、canonical 导出和字节级往返。
- 当前 `App.tsx` 约 390 行，已承担导入、播放、选择和导出；P2 不继续把编辑状态、
  命令历史和时间轴操作堆入该组件。

P1 已知但不阻断 P2 的事项：

- 编辑器构建约 1.48 MB，主要来自 svgcanvas；P2 在 UI 稳定后做一次代码分包检查，
  但不以牺牲可维护性换取体积数字。
- P1 未接入生产 Tauri 播放器；该工作属于 P3。
- 自动保存、崩溃恢复、最近项目、第二套服装和发布安装包属于 P4/P5。

## 3. P2 范围

### 3.1 必须完成

- 可折叠 Part 树、可见性、锁定和舞台双向选择。
- Clip 列表和 Clip 元数据编辑。
- 帧标尺、播放头、Part 轨道、属性关键帧和离散 `renderSlot` 表现。
- 单关键帧与多关键帧选择。
- `F6` 插入/更新，`Shift+F6` 删除，复制、粘贴、拖动和批量平移。
- Transform 属性面板、pivot 工具、预设缓动和 cubic-bezier 数值编辑。
- 舞台平移、旋转和缩放手柄。
- 播放、暂停、首尾帧、逐帧和循环预览。
- 统一的项目命令历史、撤销、重做和未保存提示。
- rig/motions 分别导入、校验和 canonical 导出。
- 完整的纯状态测试、适配层测试和真实浏览器验收。

### 3.2 明确不做

- 不修改桌宠 `TianyiPet`、Tauri 窗口、点击穿透、拖动、托盘或设置页。
- 不把编辑器嵌入生产桌宠窗口。
- 不在 P2 替换当前生产 CSS 招手；该迁移属于 P3。
- 不实现路径节点编辑、画笔、形状创建、布尔运算或完整 Inkscape 替代品。
- 不允许编辑器删除或改写源 SVG 几何。
- 不实现 IK、蒙皮、网格变形、骨骼权重或任意动作重定向。
- 不实现任意逻辑父级重绑。P2 只展示 v1 rig 已声明的父子结构；当前小洛宝保持扁平 rig。
- 不实现事件标记编辑器。P2 必须无损保留既有 `events`，事件创作留到后续阶段。
- 不实现自动保存、崩溃恢复、最近项目和原生文件系统写入。
- 不新增动作格式字段，不创建“P2 私有 motions 格式”。

## 4. 核心架构决策

### 4.1 单一生产数据源

React 编辑器状态中的 `CharacterRigV1` 和 `MotionLibraryV1` 是唯一生产数据源。svgcanvas DOM
只是素材显示、命中和即时预览的投影，不能反向成为保存事实。

```text
EditorDocument (rig + motions)
              │
              ├── sampleMotionClip() ──► 舞台预览
              ├── TimelineView       ──► 关键帧 UI
              ├── Inspector          ──► 属性 UI
              └── canonical export   ──► rig.json / motions.json
```

以下内容不得放进生产 JSON：

- 当前选择、播放头、面板折叠状态和时间轴缩放。
- Part 的编辑器临时显隐和锁定。
- 正在拖动的预览草稿。
- 撤销栈和剪贴板。
- svgcanvas DOM 节点、选择器或内部对象。

### 4.2 SVG 几何保持只读

P2 将 svgcanvas 自身的几何撤销栈清空并保持禁用，不允许形成“svgcanvas 一套历史、动作编辑器
另一套历史”。所有舞台变换最终都转成 rig 或 motion 命令，舞台 DOM 随当前文档重新投影。

允许使用的 svgcanvas 能力：

- `setSvgString()` / `getSvgRoot()`：载入和访问安全素材。
- `selectOnly()` / `clearSelection()`：同步语义 Part 选择。
- `bind()` / `unbind()`：接收公开的选择和画布事件。
- `setZoom()` / `getZoom()`：舞台缩放。

禁止主路径依赖 `getPrivateMethods()`、私有 selector 对象、内部 history command 或 SVG-edit
完整 UI 状态。

### 4.3 使用项目自有 Transform Gizmo

svgcanvas 默认旋转手柄围绕图形选择框中心，不能保证使用 rig 中的 Part-local pivot。P2 不直接
把默认选择框变换作为动作数据来源，而是在舞台上叠加项目自己的轻量手柄：

- 部件主体拖动：编辑 `x/y`。
- pivot 到旋转手柄：编辑 `rotation`。
- 四角手柄：等比缩放；边中手柄：分别编辑 `scaleX/scaleY`。
- 独立 pivot 工具：编辑 `rig.parts[].pivot`。
- 手柄只消费适配层公开的屏幕几何和坐标换算，不进入源 SVG，不参与导出。

这套手柄必须围绕真实 pivot 计算，并在 100%、150% 和 200% DPR 下保持一致。若 P2-0
尖峰无法稳定完成坐标换算，应停止时间轴 UI 扩展，先解决手柄与舞台坐标契约。

### 4.4 没有关键帧时不静默写入

默认关闭自动打帧。当前帧没有所选 Part 的关键帧时：

- 属性面板显示当前采样姿态，但生产字段不可直接提交。
- 舞台变换手柄显示，但开始变换前提示先按 `F6` 或点击“在此帧插入关键帧”。
- 插入后才允许提交变换。

`F6` 插入的是当前采样姿态的完整数值快照：`x`、`y`、`rotation`、`scaleX`、
`scaleY` 和 `opacity`。`renderSlot` 只有在用户明确添加或修改层级关键帧时写入，避免无意
覆盖离散绘制顺序。

### 4.5 一个手势只产生一条历史记录

拖动播放头不进入历史。Part 或关键帧拖动时：

1. `pointerdown` 捕获开始文档值。
2. `pointermove` 只更新临时 draft 和舞台预览。
3. `pointerup` 将最终值作为一条命令提交。
4. `pointercancel`、Escape 或组件卸载丢弃 draft，恢复提交前状态。

因此连续 200 次 pointermove 不能制造 200 条 undo 记录。

## 5. 状态模型

建议将 P2 状态拆为三个层次：

```ts
interface EditorDocument {
  rig: CharacterRigV1;
  motions: MotionLibraryV1;
}

interface EditorSession {
  activeClipId: string | null;
  currentFrame: number;
  selectedPartId: string | null;
  selectedKeyframes: KeyframeRef[];
  hiddenPartIds: Set<string>;
  lockedPartIds: Set<string>;
  expandedPartIds: Set<string>;
  expandedTrackIds: Set<string>;
  timelinePixelsPerFrame: number;
  tool: "select" | "pivot";
  isPlaying: boolean;
}

interface EditorDraft {
  transform?: TransformGestureDraft;
  keyframeMove?: KeyframeMoveDraft;
  pivot?: PivotGestureDraft;
}
```

约束：

- `EditorDocument` 可校验、可序列化、可进入历史。
- `EditorSession` 不进入 motion/rig 导出，也不进入撤销历史。
- `EditorDraft` 只能存在于一个手势期间，不能跨 Clip、导入、撤销或播放保留。
- imported DOM refs 只保存在 `SvgCanvasAdapter`，不进入 React state 和历史快照。
- 更新采用不可变纯函数，不直接修改已提交的 rig/motions 对象。

## 6. 命令历史与脏状态

### 6.1 统一命令类型

所有会改变导出结果的操作必须通过统一命令入口，例如：

```ts
type EditorCommand =
  | { type: "clip.add"; clip: MotionClipV1 }
  | { type: "clip.updateMeta"; clipId: string; patch: ClipMetaPatch }
  | { type: "clip.delete"; clipId: string }
  | { type: "keyframe.upsert"; clipId: string; partId: string; keyframe: MotionKeyframeV1 }
  | { type: "keyframe.deleteMany"; refs: KeyframeRef[] }
  | { type: "keyframe.moveMany"; refs: KeyframeRef[]; deltaFrames: number }
  | { type: "keyframe.paste"; targetFrame: number; payload: KeyframeClipboard }
  | { type: "rig.updatePivot"; partId: string; x: number; y: number };
```

命令 reducer 必须是纯 TypeScript，不 import React、DOM 或 svgcanvas。首版数据量很小，历史
可以保存不可变 `EditorDocument` 快照，默认上限 200 条；先保证正确性，不提前实现复杂 patch
压缩。超过上限丢弃最旧记录，但不得丢弃当前保存基线判断。

### 6.2 撤销与重做

- 新命令提交后清空 redo 分支。
- 导入新 rig/motions 后建立新的历史根，不允许撤销回上一项目。
- 撤销或重做期间停止播放并取消手势 draft。
- Clip 删除、pivot 修改、多关键帧移动都必须一次撤回。
- Part 显隐、锁定、面板折叠和播放头不进入 undo。

### 6.3 未保存提示

使用 `serializeRig()` 和 `serializeMotionLibrary()` 的 canonical 文本组成保存基线。当前文档与
基线不同即为 dirty；撤销回基线时必须自动恢复为 clean。

- 成功载入后建立基线。
- 用户明确导出 rig 和 motions 后，只有两个当前版本都导出成功才更新完整基线。
- 替换项目、重新载入素材或关闭页面时，如果 dirty，显示明确确认。
- 使用 `beforeunload` 作为浏览器关闭兜底，但不声称已经实现 P4 的自动恢复。

## 7. 时间轴语义

### 7.1 布局

```text
┌─────────────┬─────────────────────────────┬──────────────┐
│ Part / Rig  │            舞台             │ 属性 / Pivot │
│ 显隐与锁定  │        Transform Gizmo      │ 缓动 / Slot  │
├─────────────┴─────────────────────────────┴──────────────┤
│ Clip │播放│帧标尺与播放头                                  │
│ Part/属性轨道 │ ◆──────◆──────◆                           │
└───────────────────────────────────────────────────────────┘
```

时间轴至少保留 280px 高度，可垂直滚动；轨道名称区固定，帧区横向滚动。播放头和帧标尺
共用一个 `pixelsPerFrame`，不能各自进行四舍五入。

### 7.2 轨道

顶层按 Part 展示，展开后展示：

- `position`（合并显示 x/y 标记，属性面板仍分别编辑）
- `rotation`
- `scale`（合并显示 scaleX/scaleY）
- `opacity`
- `renderSlot`（离散阶梯轨道）

同一 Part 同一 frame 在 v1 数据中只有一个 keyframe 对象。UI 可以在多个属性行绘制标记，
但复制、拖动和删除必须按“关键帧对象 + 属性集合”处理，不能制造重复 frame。

### 7.3 帧与播放边界

- 合法范围为闭区间 `0..durationFrames`。
- repeat 播放超过末帧后回到 0；手动定位必须能检查末帧。
- 修改 duration 导致关键帧或 event 越界时拒绝提交，并列出阻塞项；不静默裁剪。
- FPS 支持 schema 范围内整数，修改 FPS 不自动重采样帧号。
- 播放采用 `requestAnimationFrame` 和时间戳换算，不通过 interval 累加帧。

### 7.4 关键帧选择与拖动

- 单击选择一个关键帧；`Ctrl` 增减选择；`Shift` 选择同轨道连续范围。
- 多选可以跨 Part，但拖动时全部使用相同整数 `deltaFrames`。
- 拖动结果不得小于 0、超过 duration，也不得与目标轨道已有 frame 冲突。
- 冲突时保持最后一个合法预览位置并显示原因；松手不覆盖既有关键帧。
- 拖动时使用 pointer capture，支持窗口外松手和 `pointercancel`。
- 键盘左右方向键移动选中关键帧 1 帧，`Shift+方向键` 移动 5 帧。

### 7.5 复制与粘贴

编辑器使用结构化内部剪贴板，不把未经校验的 JSON 直接放入系统剪贴板：

- 复制记录所选关键帧相对最早 frame 的偏移、Part ID 和值。
- 粘贴以当前播放头为锚点。
- 粘贴前校验 Part、范围、重复 frame 和 schema。
- 有冲突时整次粘贴拒绝，不做半成功写入。
- P2 不要求跨浏览器标签页或跨项目粘贴。

## 8. Part 树与舞台选择

### 8.1 Part 树

- 树结构从 `logicalParentId` 派生；当前扁平 rig 合法显示为多个根节点。
- 同层稳定顺序按 `renderSlots` 顺序，再按 rig 中已有 Part 顺序。
- 眼睛图标切换编辑器可见性；锁图标切换编辑器锁定。
- 隐藏只影响舞台，不修改 motion opacity。
- 锁定部件仍显示，但不能从舞台选择或变换；仍可从树中查看并解锁。
- 选中 Part 后自动展开对应轨道，并在属性面板显示当前采样值。

### 8.2 双向选择

- Part 树选择调用 `adapter.selectPart(partId)`。
- 舞台点击通过 DOM 祖先映射到语义 Part，再更新编辑器选择。
- 点击 Part 内部 path 时必须归一化到整个 Part，不能进入内部路径节点编辑。
- pivot 标记和编辑器 overlay 永远不能被识别为角色 Part。
- 程序化选择必须使用重入保护，避免 `selectOnly()` 与 `selected` 回调循环。

## 9. 属性面板

### 9.1 Transform

显示并编辑：

- `x`、`y`
- `rotation`（度）
- `scaleX`、`scaleY`
- `opacity`（UI 可显示百分比，数据仍为 0..1）

输入规则：

- 输入过程中允许暂时为空；只在 Enter、blur 或步进按钮时提交合法有限数值。
- Escape 恢复提交前值。
- scale 不允许落到 schema 禁止范围；近零矩阵必须明确报错。
- 多字段一次提交可合并为一条历史命令。
- 当前帧无关键帧时显示“采样值”，并提供明确的插入按钮。

### 9.2 Pivot

- 显示 Part-local `x/y`。
- 通过数值或 pivot 工具修改。
- 修改进入 rig 历史，不写入 motion。
- 有 rotation/scale 关键帧的 Part 修改 pivot 时显示影响提示；P2 不自动改写所有动作补偿位移。
- pivot 拖动、撤销、导出 rig、重新导入后位置必须一致。

### 9.3 Easing

- 提供 `linear`、`easeIn`、`easeOut`、`easeInOut`。
- 提供四个 cubic-bezier 数值输入，x 控制点限制在 0..1。
- easing 属于当前关键帧到下一个含对应属性关键帧的出段，UI 必须使用“出段缓动”措辞。
- 末关键帧允许保留 easing，但提示它当前没有可见出段。

### 9.4 Render Slot

- 值只能来自 `rig.renderSlots`。
- 以离散关键帧保存，不做插值。
- 默认不随 `F6` 自动写入。
- 删除最后一个 slot 值后恢复 `defaultRenderSlot`。

## 10. Clip 管理

P2 动作列表至少支持：

- 新建：填写合法唯一 ID、FPS、duration 和 loop。
- 复制：复制全部轨道、事件和 suppressProceduralChannels，生成新 ID。
- 重命名：只改 clip ID，不改变动作内容。
- 删除：需要确认；动作库至少保留一个 clip，或清晰支持空动作库并禁用时间轴。
- 编辑元数据：FPS、duration、loop 和 `suppressProceduralChannels`。

默认模板：

| Clip | FPS | 时长 | Loop | 用途 |
|---|---:|---:|---|---|
| `idle` | 24 | 96 帧 | repeat | 小幅身体呼吸或轻摆 |
| `wave` | 24 | 24 帧 | none | 右手挥动一次 |
| 自选动作 | 24 | 用户指定 | none/repeat | 验证非硬编码创作 |

不能通过 `createWaveExample()` 的硬编码结果冒充用户制作闭环；该按钮可以继续作为教程样例，
但最终验收的 `wave` 必须能从空 Clip 经 UI 制作。

## 11. 快捷键与焦点

| 快捷键 | 行为 |
|---|---|
| `F6` | 插入或更新所选 Part 当前帧关键帧 |
| `Shift+F6` | 删除所选 Part 当前帧关键帧 |
| `Enter` | 播放/暂停 |
| `,` / `.` | 前一帧/后一帧 |
| `Home` / `End` | 首帧/末帧 |
| `Ctrl+C` / `Ctrl+V` | 复制/粘贴关键帧 |
| `Delete` | 删除选中关键帧 |
| `Ctrl+Z` | 撤销 |
| `Ctrl+Shift+Z` / `Ctrl+Y` | 重做 |
| `Escape` | 取消当前手势或草稿 |
| `Space + 拖动` | 平移舞台 |

约束：

- `input`、`textarea`、`select` 和 contenteditable 聚焦时，除 Escape 外不抢占文本编辑快捷键。
- 浏览器保留快捷键需要按作用域谨慎阻止默认行为。
- 每个快捷键必须有可点击按钮或菜单替代路径。
- 按钮需要中文 `aria-label`；关键帧必须可以获得键盘焦点。

## 12. 共享核心增量

P2 可以向 `@ltypet/character-motion` 增加通用纯函数，但不得加入 React 或 DOM 依赖。预计需要：

- 从 `bindMatrix⁻¹ × editedLocalMatrix` 恢复 authored transform 的无 skew 分解。
- 围绕 pivot 分解 `x/y/rotation/scaleX/scaleY`。
- 检测 skew、奇异矩阵和数值误差。
- 关键帧 upsert、删除、批量移动、冲突检测和范围检查。
- Clip ID 唯一性和动作编辑后校验辅助。

若这些函数只服务 UI 选择和像素布局，应留在编辑器；只有格式语义、矩阵语义和时间轴语义
才能进入共享包。

P2 不修改 v1 schema。若实现过程中发现必须新增格式字段，应暂停并单独形成 schema 变更决策，
不能在 UI 代码中偷偷写入额外字段。

## 14. 依赖策略

P2 默认不新增依赖，现有 React、Vitest、jsdom 和 svgcanvas 足以完成首版：

- 时间轴拖动使用 Pointer Events，不引入通用 DnD 框架。
- 快捷键使用集中式 hook，不引入快捷键库。
- UI 使用原生 CSS Grid/Flex，不引入完整组件库。
- 浏览器验收继续使用本机 Edge/CDP 或手工验收，不把大型 E2E 框架提前加入 P2。

只有出现已有方案无法可靠测试的具体阻塞，才可以评估增加测试辅助依赖；增加前必须记录
`node --version`、`npm ls --depth=0`，并说明依赖解决的具体问题。根目录仍忽略
`package-lock.json`，不得假定不同机器会得到相同宽松小版本。

## 20. 风险与停止条件

| 风险 | 处理方式 |
|---|---|
| svgcanvas 默认手柄不使用 rig pivot | 使用项目自有 Transform Gizmo，不接入私有 selector |
| svgcanvas 点击选中内部 path | 适配层向上归一化到语义 Part，并加重入保护 |
| DOM 被当成保存事实 | 所有提交走 EditorDocument，DOM 只做投影 |
| 拖动产生大量 undo | draft 预览，pointerup 单命令提交 |
| pivot 修改改变既有动作外观 | 明确提示、允许撤销；P2 不做全库自动补偿 |
| 多属性 UI 与单 keyframe 数据冲突 | 以 frame 上单一 keyframe 对象为事实，属性行只是视图 |
| duration 缩短丢数据 | 有越界 keyframe/event 时拒绝，不静默裁剪 |
| 两套撤销栈互相打架 | P2 源 SVG 几何只读，svgcanvas history 不作为主路径 |
| UI 状态误入生产格式 | EditorSession/EditorDraft 与 v1 文档类型分离 |
| P2 范围膨胀成完整绘图软件 | 路径、形状、IK、自动恢复和服装验证留在既定后续阶段 |

出现以下任一情况时，应暂停继续堆 UI，并先回到架构决策：

- 必须调用 `getPrivateMethods()` 才能完成核心选择或变换。
- 真实 pivot 在任一目标 DPR 下持续漂移且无法通过公开坐标契约修复。
- 舞台变换不能无损恢复为 v1 authored transform。
- 一次手势无法形成单一可撤销命令。
- 导出后重新导入无法逐帧得到同一姿态。
